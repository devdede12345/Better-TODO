# Better TODO — Obsidian Plugin

Port of the standalone Better TODO app to an Obsidian plugin.

## Features (initial scaffold)

- **Strikethrough decorations** for completed (`✔` / `- [x]`) and cancelled
  (`✘` / `- [-]` / `- [/]`) tasks in live preview, source mode and reading
  view
- **`@due(...)` highlighting** — overdue items get a red badge in the editor
- **`@lasted(...)` / `@done(...)` tag styling**
- **Timeline view** — registered as a sidebar pane (`calendar-clock` ribbon
  icon or command palette `Better TODO: Open Timeline view`)
- **Commands**:
  - `Better TODO: Toggle task: done`
  - `Better TODO: Toggle task: cancelled`
  - `Better TODO: Archive completed tasks`
- Dual-syntax support: native Markdown task lists **and** the legacy `.todo`
  glyph syntax (`☐ ✔ ✘`)

## Development

From `obsidian-plugin/`:

```bash
npm install
npm run dev          # builds CSS once, then watches JS via esbuild
# or, in two terminals:
npm run dev:css      # tailwind watch
npm run dev:js       # esbuild watch
```

Production:

```bash
npm run build        # outputs main.js + styles.css to repo root
```

## Installing into a vault

1. Build (or run `npm run build`).
2. Copy `manifest.json`, `main.js`, and `styles.css` into
   `<your-vault>/.obsidian/plugins/better-todo/`.
3. In Obsidian → Settings → Community plugins → enable **Better TODO**.

For local iteration, symlink the folder instead of copying:

```powershell
# PowerShell (admin)
New-Item -ItemType Junction `
  -Path "$env:APPDATA\obsidian\<vault>\.obsidian\plugins\better-todo" `
  -Target "C:\Github\Better-TODO\obsidian-plugin"
```

## Caveats

- The Timeline UI reuses the standalone app's Tailwind classes verbatim.
  Tailwind preflight is **disabled** in the plugin build to avoid resetting
  Obsidian's own styles, but generated utility classes are global. If you
  hit visual conflicts, narrow Tailwind's scope by enabling `important:
  ".btodo-root"` in `tailwind.config.cjs`.
- Spotlight search, file explorer, settings panel, sticker window, and
  custom keymap from the standalone app are **not** ported in this
  scaffold — they map awkwardly to Obsidian's existing UX. If you want
  them brought over (e.g. as a custom modal), open an issue.
